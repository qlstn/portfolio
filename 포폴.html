치매 진단 AI 모델 개발

2024.03 - 2024.06팀 프로젝트역할: 조장 / 데이터 핸들링

문제의식

고령화로 치매 환자가 증가하면서, 건강 데이터 기반으로 치매 발생 가능성과 관련 요인을 분석할 필요가 있다고 판단했습니다.


내 역할

매주 회의를 열어 진도를 확인하고 역할을 조율했습니다. 데이터 전처리, 결측치 처리, 변수 선택, 스케일링, 데이터 분할 과정을 담당했습니다.

결과

  • R²: 0.692007
  • Adjusted R²: 0.679107
  • MSE: 0.1
  • 교차검증 R²: 약 0.7

배운점

데이터의 미세한 변화가 모델 결과에 영향을 줄 수 있다는 점을 배웠습니다. 특히 결측치 처리, 변수 선택, 데이터 분할처럼 기본 과정이 정확해야 결과를 신뢰할 수 있다는 것을 경험했습니다.

AI 기반 IoT 악성코드 탐지

2025.09 - 2025.12개인 프로젝트역할: 전체 과정 단독 수행

문제의식

IoT 악성 트래픽은 단일 패킷보다 흐름과 시간적 패턴에서 탐지 단서가 드러날 수 있다고 보고, PCAP 데이터를 시퀀스 형태로 구성해 모델별 탐지 성능을 비교했습니다.


내 역할

PCAP 특징 추출, Flow 단위 그룹화, Sliding Window 적용, 수치형/범주형 데이터 전처리, GRU와 Transformer 모델 학습 및 성능 비교를 모두 수행했습니다.

결과

  • GRU: Binary Accuracy 0.9626, Multi Accuracy 0.9470
  • Transformer 2-layer: Binary Accuracy 0.9875, Multi Accuracy 0.7944
  • Transformer 12-layer: Binary Accuracy 0.9813, Multi Accuracy 0.9595
  • 최종 비교 F1: GRU Multi 0.945, Transformer 12-layer Multi 0.95

배운점

전체 정확도만으로 모델을 판단하면 부족하다는 점을 배웠습니다. 특정 클래스의 성능 저하를 확인하고, 데이터 불균형을 완화하기 위해 클래스 가중치, 오버샘플링, Focal Loss를 적용했습니다.

Docker 기반 AI 모델 이상 요청 탐지 및 차단

2025.09 - 2025.12팀 프로젝트역할: 로그 기반 탐지 구조 구현 및 검증

문제의식

LLM 서비스는 Prompt Injection, Command Injection, Information Leak과 같은 입력 기반 공격에 노출될 수 있습니다. 정적 Rule만으로는 우회형 요청 탐지에 한계가 있어 로그 기반 ML 탐지를 결합했습니다.


내 역할

Llama 모델을 Docker 환경에서 실행하고, 정상/악성 요청 시드 기반 로그를 수집했습니다. proxy_access.log를 5초 단위로 집계해 특징을 만들고 RandomForest 탐지를 적용했습니다.

결과

  • Request Generator: benign 100개, malicious 100개 시드 사용
  • 수집 로그: 총 700개
  • 학습 데이터: 약 680개 로그
  • Accuracy: 약 85%
  • F1-score: 0.79
  • 악성 요청 차단 시 403 반환

배운점

프롬프트와 응답을 단순 문장으로 보지 않고, 요청의 의도와 패턴을 함께 확인해야 한다는 점을 배웠습니다. 또한 Rule 기반 탐지와 ML 기반 탐지를 결합하면 우회형 공격 대응 가능성을 높일 수 있음을 확인했습니다.

Wipe Engine 개인정보 흔적 제거 솔루션

2025.12팀 프로젝트역할: 카카오톡 아티팩트 분석 및 삭제 기능

문제의식

공용 PC에서 브라우저나 메신저 사용 후 로그아웃하지 않거나 파일을 삭제하지 않으면, 다음 사용자가 이전 사용자의 자동 로그인 정보와 사용 흔적에 접근할 수 있습니다.


내 역할

카카오톡의 로컬 로그인 정보, 사용자 데이터, 다운로드 파일, 캐시 및 프로필 관련 데이터가 남는 구조를 분석하고 삭제 흐름을 정리했습니다.

결과

  • Chrome, KakaoTalk, Windows 3개 영역 제거 기능 구현
  • KakaoTalk: 로그인 정보, 받은 파일, 채팅 기록 및 프로필 정보 3개 범주 정리
  • login_list.dat, last_pc_login.dat 사용자 ID값 무효화 흐름 설계
  • chatLog, talkuserdb 등 캐시/DB 관련 파일 덮어쓰기 후 삭제 흐름 정리

정량 성능지표가 아닌 기능 구현 및 제거 결과 중심 프로젝트입니다.

배운점

단순 파일 삭제와 개인정보 흔적 제거는 다르다는 점을 배웠습니다. 사용자가 인식하지 못하는 시스템/앱 아티팩트를 분석하고, 실제 사용자의 행동 패턴을 고려해 기능을 설계하는 경험을 했습니다.

MCP 서버 취약점 탐지 및 악용 가능성 검증

2026.03 - 2026.05캡스톤/논문 프로젝트역할: 테스트 케이스 구성 및 실험 수행

문제의식

정적 분석은 취약점 후보를 찾을 수 있지만, 실제 MCP 호출 과정에서 악용 가능한지까지 판단하기 어렵습니다. 이를 보완하기 위해 실행 기반 검증을 결합했습니다.


내 역할

Filesystem, Git, Memory 서버를 대상으로 positive 30개와 negative 30개 테스트 케이스를 구성하고, 정적 분석과 하이브리드 분석 결과를 비교했습니다.

결과

  • 평가 데이터셋: positive 30개 + negative 30개 = 총 60개
  • 정적 분석: TP 22, FP 20, FN 8
  • 정적 분석 Precision 0.524, Recall 0.733, F1 0.611
  • 하이브리드 분석: Static Findings 42건 중 Confirmed 29건
  • 하이브리드 Precision 0.690

배운점

탐지 결과를 그대로 신뢰하지 않고, 기준과 근거를 바탕으로 다시 확인하는 과정이 중요하다는 것을 배웠습니다. 테스트 케이스와 평가 지표는 결과 품질을 설명하는 핵심 근거가 된다는 점을 경험했습니다.
